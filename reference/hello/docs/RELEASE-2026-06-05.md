## Features shipped

- **S16 Admin data views** — Customers / Store / InvoxAI-TR / Domains (read-only).
- **S17 Admin ops** — System Health, Broadcast, in-house Support tickets.
- **S18 Discord integration** — paid Discord access (invite-link model), migration `057`, `discord-expiries` cron.
- **S1B** — multi-gateway drivers + refund-through-own-gateway fix.
- **S19 teardown** — dropped the dormant payout/KYC schema. Migration `058` (CSV backup taken first).
- **Gap-fill** — booking `.ics` + reschedule, buyer-portal bookings, generic gateway webhook, Discord interactions endpoint, `testPixelAction`, `setSendingDomainAction`. Migration `059`.

### Store (Phase 2)
- **Phase 2a** — catalog products + collections (each catalog product auto-creates a published payment page). Migration `061`.
- **Phase 2b — multi-item cart** — `order_items` (migration `062`), one-seller-per-cart, separate `create-cart-order` / `verify-cart-payment`, shared `fulfillCartOrder`. Cart-order line items on `/order/[id]` + cart invoices.

### Booking
- **S11 group events** — fixed date/time, many attendees up to a capacity (free or paid). `booking_events` + `event_registrations` + atomic `register_for_event` RPC (row-locked; stale pending holds expire after 15 min). Migration `064`.

### Support
- **S17 live chat** — the ticket thread updates live via short-poll (`/api/support/[id]/messages`, server-auth). Websocket Realtime deferred to v2.

### Video DRM (S9, zero-cost)
- **Forensic watermark** — per-viewer email tiled across the whole frame (unccroppable, traceable).
- **True HLS + AES-128** — uploads transcode (BullMQ `hls` worker, system ffmpeg) to encrypted HLS; key served only to authorized viewers via `/api/courses/hls/{playlist,key}`; hls.js player. Migration `065`. Raw MP4 deleted after a successful transcode.
- **Admin Video-DRM view** — `/admin/transcodes` lists transcode status + retry failed ones.

## Bugs caught & fixed before they bit (during audits/QA)
1. **Cart `source` CHECK constraint** — `create-cart-order` inserted `source='cart'`, which the live `orders_source_check` rejected → every cart checkout would have 500'd. Caught by post-deploy live QA; fixed by migration `063`.
2. **verify-payment double-charge** (pre-existing) — financial side-effects weren't gated on the pending→paid transition → racing/retried confirms double-charged the wallet fee + double-decremented stock. Fixed with the `didTransition` guard + migration `060`.
3. **Course-video access leak** (pre-existing, CRITICAL) — the course token was not scoped to the video, so any valid token could fetch any seller's course video / HLS key. Fixed in `lib/courses/media-auth` (binds token.course_id → the lesson) — also patched the live `video-url` route.

Plus smaller fixes: `.ics` PII removal, marketing-webhook SSRF guard, buyer OTP rate-limit, event capacity/orphan-order edges.
